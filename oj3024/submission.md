# บันทึกการแก้โจทย์

ไฟล์นี้ต้องเขียนโดยนักศึกษาด้วยคำพูดของตนเอง

ใช้ template นี้เฉพาะกับโจทย์ OJ ที่ถูกระบุว่าต้องส่ง learning log เท่านั้น

ห้ามให้ AI เขียนไฟล์นี้แทนคุณ AI อาจช่วยตรวจ grammar, formatting หรือความชัดเจนได้ หลังจากที่คุณเขียนเนื้อหาของตนเองแล้ว

ถ้าใช้ AI กับโจทย์ learning-log-required นี้ ต้องทำไฟล์ `ai_reflection.md` ด้วย

- ถ้าใช้ทั้ง AI และความช่วยเหลือจากคน ให้นับเฉพาะเวลาที่ทำด้วยตนเองก่อนขอความช่วยเหลือจากภายนอกครั้งแรก ไม่ว่าจะเป็น AI หรือคน
- ถ้าไม่ได้ใช้ AI และไม่ได้ขอความช่วยเหลือจากคน ให้นับเวลาถึงก่อนเขียน `submission.md`
- ประมาณเวลาได้ แต่ต้องซื่อสัตย์

## 2. ความเข้าใจโจทย์ของฉัน

เขียนโจทย์ด้วยคำพูดของตนเอง

ให้อธิบาย input, output และ constraints สำคัญด้วย

ถ้ายังไม่เข้าใจโจทย์ทั้งหมด ให้เขียนสิ่งที่เข้าใจในตอนนี้ ความเข้าใจอาจยังไม่ครบหรืออาจผิดได้ แต่ต้องพยายามอธิบายอย่างจริงใจ

```text
โจทย์ต้องการให้ฉันหาว่ามีโหวตแปลกๆ หรือไม่ที่ทำให้คะแนนผลโหวตอันดับสูงสุดมีค่าห่างจากคะแนนผลโหวตอื่นๆมากกว่า 2

Input: คะแนนรวมของผลโหวตสามคน และคะแนนสูงสุดของผลโหวต
Output: หาว่ามีหน้าม้ามาปั่นโหวตให้คนที่มีผลโหวตคะแนนสูงสุด คะแนนมากกว่าคะแนนรองลงมา มากกว่า 2 คะแนน ถ้าใช่แสดงผลลัพธ์ Surprising ถ้าไม่ แสดงผลลัพธ์ Not surprising
Constraints: ต้องหาว่าอีกสองคนที่เหลือมีผลโหวตห่างจาก อันดับสูงสุดโดยที่จำนวนผลโหวตของแต่ละอันดับต้องไม่เกิน 10 ทำให้ผลรวมมีค่าที่มากที่สุดคือ 30 และต้องระวังในกรณีที่ผลโหวตเท่ากับ 0 หรือติดลบด้วย
```

## 3. แผนแรกของฉัน

เขียนแผนแรกก่อนรับความช่วยเหลือจาก AI เพื่อน TA ผู้สอน หรือก่อนสรุป code สุดท้าย

ถ้าใช้ AI ให้เขียนแผนที่มีอยู่ก่อน prompt แรกที่ถาม AI

ถ้าถามเพื่อน TA หรือผู้สอน ให้เขียนแผนที่มีอยู่ก่อนถาม

ถ้าไม่ได้ใช้ AI และไม่ได้ขอความช่วยเหลือจากคน ให้เขียนแผนที่มีอยู่ก่อนหรือระหว่างเริ่มเขียน code

แผนนี้เขียนแบบคร่าว ๆ ได้ อาจยังไม่สมบูรณ์หรืออาจต่างจากวิธีสุดท้าย

สามารถเขียนเป็น pseudocode, flowchart idea หรือขั้นตอนความคิดได้

```text
step1:รับค่าคะแนนทั้งหมด
step2:รับค่าคะแนนสูงสุด
step3:หาผลคะแนนที่เหลือจากการนำ คะแนนทั้งหมด - คะแนนสูงสุด
step4:เนื่องจากเหลือเพียง 2 อันดับรองลงมา จึงนำคะแนนที่เหลือไปหาร 2
step5:ทำการคำนวณหาความต่างของคะแนน
step6:ถ้าความต่างของคะแนนที่มากที่สุดลบกับน้อยที่สุดน้อยกว่าสอง ให้พิมพ์ "Not surprising"
step7 :นอกเหนือจากนั้นพิมพ์ "Surprising"
```

## 4. วิธีสุดท้ายที่ใช้จริง

อธิบายสั้น ๆ ว่า algorithm หรือวิธีสุดท้ายที่ใช้จริงใน code ที่ส่งคืออะไร

หัวข้อนี้ต่างจาก Section 3:

- Section 3 คือแผนแรกก่อนใช้ AI ก่อนรับความช่วยเหลือจากคน หรือก่อนเขียน code สุดท้าย
- Section 4 คือวิธีสุดท้ายที่ใช้ใน solution จริง
- ถ้าวิธีสุดท้ายเหมือนกับแผนแรก ให้เขียนว่าเหมือนกัน และอธิบายสั้น ๆ ว่าทำไม

ห้ามคัดลอกคำอธิบายจาก AI

ห้ามคัดลอกคำอธิบายจากคนอื่น

```text
วิธีสุดท้ายที่ใช้จริง ฉันมีการปรับแผนตามคำแนะนำของ AI โดยที่ปรับจากการนำจำนวนมากสุดไปลบสองอันดับที่เหลือหารด้วย 2 มาเป็นการที่ใช้คะแนนรวมกัน ลบด้วยคะแนนมากสุดสองครั้ง เพื่อที่จะหาค่าที่น้อยที่สุดของคะแนนในการโหวตครั้งนี้ จากนั้นฉันปรับ workflow การทำงานของ if-else และเพิ่มเงื่อนไขที่คะแนนต่ำสุดน้อยกว่า 0 ลงไป
```

## 5. การทดสอบของฉัน

เขียน test cases อย่างน้อย 3 กรณีที่ลองเองหรือออกแบบเอง

พยายามเลือก test cases ที่แตกต่างกัน

แต่ละ test case ให้อธิบายว่าทำไมเลือกกรณีนั้น

ถ้า input หรือ output มีหลายบรรทัด ให้เขียนไว้ใน text blocks

### Test Case 1

ทำไมเลือก case นี้:

```text
เพราะต้องการทดสอบว่า หากผลรวมการโหวตเป็น 0 คะแนนมากสุดยังเป็น 0 จะต้องลงเงื่อนไข not surprising เพราะค่ามากสุดและน้อยสุดห่างกันน้อยกว่า 2
```

Input:

```text
0
0
```

Expected output:

```text
Not surprising
```

Actual output:

```text
Not surprising
```

Result:

```text
Pass
```

### Test Case 2

ทำไมเลือก case นี้:

```text
เพราะว่าต้องการทดสอบในกรณีที่ทั้งสามคนมีผลโหวตเท่ากันหมดเลย ซึ่งความต่างของค่ามากและค่าน้อยที่สุดเท่ากับ 0 พอดี
```

Input:

```text

6
```

Expected output:

```text
Not surprising
```

Actual output:

```text
Not surprising
```

Result:

```text
Pass
```

### Test Case 3

ทำไมเลือก case นี้:

```text
เพราะต้องการทดสอบเงื่อนไขในกรณีที่ผลโหวตติดลบ จาก testcase นี้ เพราะว่าเราหา min score จากการที่ลบผลโหวตรวม ด้วยคะแนนมากที่สุด และคะแนนอีกอันที่คาดการณ์ไว้ว่ามากที่สุดเหมือนกัน โดยที่หากคิดตรงๆเลยจะได้ติด -6 ซึ่งจะลงเงื่อนไขเดียวกับ elif ที่ min score <= 0 แต่จริงๆแล้ว min_score ควรเท่ากับ 0 เพราะคะแนนต่ำสุดที่เป็นไปได้คือ 0 ดังนั้น testcase นี้โปรแกรมควรคำนวน ออกมาได้ max = 3 และ min = 0 ทำให้ได้ผลลัพธ์เป็น Surprising
```

```text
3
3
```

```text
Surprising
```

Actual output:

```text
Surprising
```

```text
Pass
```

## 7. ความช่วยเหลือจากคน / การร่วมมือ

ได้ถามเพื่อน TA ผู้สอน หรือบุคคลอื่นเพื่อขอความช่วยเหลือในโจทย์นี้หรือไม่

```text
Yes
```

ถ้าใช่ ให้อธิบายสั้น ๆ ว่าได้รับความช่วยเหลือแบบใด

ตัวอย่างที่อนุญาต:

- อธิบายความหมายของโจทย์
- อธิบาย concept การเขียนโปรแกรม
- ให้ hint เกี่ยวกับแนวทาง
- คุยเรื่อง debugging
- คุยเรื่อง test cases
- ช่วยอธิบาย error message

สิ่งที่ไม่อนุญาต:

- คัดลอก code ของผู้อื่น
- ส่ง solution ของผู้อื่น
- ขอให้ผู้อื่นเขียน solution ให้
- ใช้ OJ submission ของผู้อื่น
- ขอให้ผู้อื่นส่ง OJ แทน

ใครช่วยคุณ

```text
เพื่อน
```

เขาช่วยอะไร

```text
แนะนำวิธีการแก้คำนวนทางคณิตศาสตร์ของข้อนี้
```

คุณยังทำอะไรด้วยตนเอง

```text
เขียนโค้ด วางแพลน และแก้ไขโค้ด
```

คุณคัดลอก code จากคนอื่นหรือไม่

```text
No
```
